# Goida VPN Site - Заметки по проекту

## 🏗 Структура проекта

- **main.py**: Основное приложение Flask.
  - Служит веб-сервером (использует `waitress` для локального хостинга).
  - Маршруты:
    - `/`: Отрисовывает главную страницу с поддержкой SEO (мета-теги, канонические ссылки).
    - `/api/download-links`: Предоставляет динамические ссылки на скачивание инструментов (v2rayNG, Throne) с кэшированием на 24 часа.
    - `/api/vc-runtime-link`: Автоматически парсит актуальную ссылку на Visual C++ Runtimes с comss.ru.
    - `/api/github-stats`: Предоставляет статистику репозитория `AvenCores/goida-vpn-configs` с кэшированием на 1 час.
    - `/robots.txt`, `/sitemap.xml`: Динамическая генерация SEO-файлов.
    - `/LICENSE`: Отдает файл лицензии.
    - `/manifest.webmanifest`, `/sw.js`: Поддержка PWA.
  - Особенности: Функция `download_badges()` для локального кэширования бэджей GitHub.

- **services.py**: Бэкенд-логика для управления конфигурациями.
  - Получает и парсит конфигурации VPN из репозитория `AvenCores/goida-vpn-configs`.
  - Извлекает из `README.md` этого репозитория временные метки последних обновлений для каждого из 26 конфигов.
  - Сопоставляет ID конфигов с их соответствующими URL-адресами источников.
  - Определяет специальные типы конфигов (рекомендованные, SNI/CIDR bypass).

- **build.py**: Скрипт сборки статического сайта и деплоя.
  - Рендерит `index.html` в директорию `dist/` с минификацией.
  - Генерирует статические JSON-файлы для API (`api/download-links.json`, `api/vc-runtime-link.json`, `api/github-stats.json`).
  - Копирует все статические ресурсы, включая переводы (`translations.json`) и файлы PWA.
  - Создает `robots.txt` и `sitemap.xml` для статической версии.
  - Управляет деплоем в ветку `gh-pages`.

- **static/**: Статические ресурсы.
  - **js/**:
    - `i18n.js`: Система интернационализации (поддержка 5 языков: RU, EN, UK, BE, DE).
    - `statistics.js`: Асинхронно загружает и отображает статистику GitHub.
    - `update-download-links.js`: Периодически обновляет ссылки на скачивание из API.
    - `link-confirmation.js`: Управляет модальными окнами подтверждения выхода и скачивания.
  - **i18n/**:
    - `translations.json`: Файл со всеми текстовыми строками на разных языках.
  - **images/**: Фавиконы, бэджи (в подпапке `badges/`) и QR-коды.
  - **css/**: `tailwind.input.css` (исходник) и `tailwind.css` (сгенерированный).

- **templates/**: Шаблоны Jinja2.
  - `base.html`: Основной макет с Alpine.js для управления состоянием (модалки, темы, мобильное меню).
  - `index.html`: Главная страница, собирающая компоненты.
  - **components/**: Модульные части UI.
    - `header.html`, `footer.html`: Шапка с переключателем языков/тем и подвал.
    - `hero.html`: Приветственная секция с бэджами статистики.
    - `tabs.html`: Основной интерфейс со списком конфигов и поиском.
    - `bypass.html`: Специальная секция для обхода SNI/CIDR.
    - `video.html`: Видео-инструкции.
    - `instructions.html`: Огромная база текстовых инструкций для всех платформ.

- **Корневые файлы**:
  - `sw.js`, `manifest.webmanifest`: Реализация Progressive Web App (PWA).
  - `tailwind.config.js`, `package.json`: Конфигурация для сборки стилей Tailwind CSS.
  - `requirements.txt`: Список зависимостей Python (Flask, requests, beautifulsoup4, waitress).

## ⚙️ Технологии
- **Бэкенд**: Python (Flask, Waitress, Requests, BeautifulSoup4).
- **Фронтенд**: TailwindCSS (JIT/CLI), Alpine.js, FontAwesome.
- **Интернационализация**: Собственная JS-реализация (i18n.js) на базе JSON-переводов.
- **PWA**: Service Workers для кэширования и возможности установки на устройства.
- **CI/CD**: GitHub Actions + `build.py` для автоматического деплоя.

## 🔑 Ключевые особенности
- **Динамические конфигурации**: Конфиги и время их обновления подтягиваются из внешнего репозитория.
- **Мультиязычность**: Полная поддержка 5 языков с сохранением выбора пользователя.
- **PWA Ready**: Сайт можно установить как приложение, он работает быстро благодаря кэшированию.
- **SEO Оптимизация**: Автоматическая генерация sitemap, robots.txt и правильные мета-теги.
- **Авто-обновление ссылок**: Ссылки на v2rayNG, Throne и VC Runtimes всегда актуальны благодаря парсингу GitHub API и comss.ru.
- **SNI/CIDR Bypass**: Отдельное внимание уделено обходу специфических блокировок.

## 📝 Обслуживание
- **Обновление конфигов**: Вносите изменения в репозиторий `AvenCores/goida-vpn-configs`.
- **Изменение текстов**: Редактируйте `static/i18n/translations.json`.
- **Стили**: При изменении `tailwind.input.css` или шаблонов запустите `npm run build:css`.
- **Деплой**: Выполните `python build.py --deploy` (требуется `MY_TOKEN` в окружении).
- **Разработка**: Используйте `python main.py --debug` для локального запуска с авто-релоадом.
